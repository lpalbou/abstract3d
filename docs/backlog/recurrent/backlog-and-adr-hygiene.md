# Recurrent: backlog and ADR hygiene

## Trigger

- After completing or deprecating backlog work
- Before release work
- After architecture-significant implementation changes

## Checklist

1. Verify `docs/backlog/overview.md` counts and links.
2. Verify backlog item filenames remain globally unique and four-digit prefixed.
3. Verify completed items include completion reports.
4. Verify accepted ADRs still match the code and docs.
5. Record any drift as a backlog or ADR update in the same pass.
