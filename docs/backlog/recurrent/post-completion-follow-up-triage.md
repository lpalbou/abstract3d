# Recurrent: post-completion follow-up triage

## Trigger

- After closing a planned backlog item that leaves residual risk, open questions, or optimization ideas

## Checklist

1. Review the completion report.
2. Classify each follow-up as planned, proposed, stale, or documentation-only.
3. Add new proposed work when the signal should survive the current task.
4. Promote directly to planned only when the risk is urgent or already well understood.
